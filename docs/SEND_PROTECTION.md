# 🛡️ حماية الإرسال المتكرر

## 📋 نظرة عامة

تم إضافة نظام حماية شامل لمنع إرسال رسائل متعددة في نفس الوقت في تطبيق Apex Sender.

## ✅ التحديثات المنفذة

### 1. متغير الحماية
```python
self.is_sending = False  # حماية من الإرسال المتكرر
```

### 2. نقاط الحماية

#### أ) في دالة `_send_files()`
- فحص قبل بدء أي عملية إرسال
- عرض رسالة تحذير للمستخدم
- منع الإرسال إذا كانت هناك عملية جارية

```python
if self.is_sending:
    QMessageBox.warning(self, "تحذير", "يوجد عملية إرسال جارية بالفعل. الرجاء الانتظار حتى تنتهي.")
    return
```

#### ب) في دالة `send_folder()`
- فحص قبل بدء ضغط المجلد
- منع الضغط والإرسال المتزامن
- تفعيل الحماية فوراً بعد اختيار المجلد

```python
if self.is_sending:
    QMessageBox.warning(self, "تحذير", "يوجد عملية إرسال جارية بالفعل. الرجاء الانتظار حتى تنتهي.")
    return

self.is_sending = True  # تفعيل الحماية
```

#### ج) في دالة `_send_files_internal()`
- فحص إضافي قبل بدء الإرسال الفعلي
- تسجيل رسالة تحذير في السجل
- منع إنشاء threads متعددة

```python
if self.is_sending:
    self.log_callback("تحذير: يوجد عملية إرسال جارية بالفعل")
    return

self.is_sending = True  # تفعيل الحماية
```

### 3. إلغاء الحماية

#### أ) عند انتهاء الإرسال بنجاح
```python
def on_finished(self, message: str, ip: str):
    self.is_sending = False  # إلغاء الحماية بعد انتهاء الإرسال
```

#### ب) عند إلغاء الإرسال
```python
def cancel_transfer(self):
    self.is_sending = False  # إلغاء الحماية عند الإلغاء
```

#### ج) عند فشل الضغط
```python
def _on_compress_error(self, error: str):
    self.is_sending = False  # إلغاء الحماية عند الفشل
```

#### د) عند إغلاق التطبيق
```python
def cleanup(self):
    self.is_sending = False  # إعادة تعيين الحماية
```

## 🔒 آلية العمل

### 1. قبل الإرسال
```
المستخدم يضغط زر الإرسال
    ↓
فحص: هل is_sending = True؟
    ↓
نعم → عرض رسالة تحذير + إيقاف
لا → تفعيل الحماية + بدء الإرسال
```

### 2. أثناء الإرسال
```
is_sending = True
    ↓
جميع أزرار الإرسال معطلة
    ↓
أي محاولة إرسال جديدة = رفض
```

### 3. بعد الإرسال
```
انتهاء الإرسال (نجاح/فشل/إلغاء)
    ↓
is_sending = False
    ↓
تفعيل أزرار الإرسال
    ↓
جاهز لإرسال جديد
```

## 🎯 السيناريوهات المحمية

### ✅ سيناريو 1: إرسال ملف أثناء إرسال آخر
```
1. المستخدم يرسل ملف A
2. أثناء الإرسال، يحاول إرسال ملف B
3. النظام يرفض ويعرض رسالة تحذير
4. بعد انتهاء A، يمكن إرسال B
```

### ✅ سيناريو 2: إرسال مجلد أثناء ضغط آخر
```
1. المستخدم يختار مجلد للضغط والإرسال
2. أثناء الضغط، يحاول اختيار مجلد آخر
3. النظام يرفض ويعرض رسالة تحذير
4. بعد انتهاء الضغط والإرسال، يمكن البدء بمجلد جديد
```

### ✅ سيناريو 3: إرسال متعدد بسرعة
```
1. المستخدم يضغط زر الإرسال عدة مرات بسرعة
2. الضغطة الأولى تبدأ الإرسال
3. الضغطات التالية تُرفض تلقائياً
4. لا يتم إنشاء threads متعددة
```

### ✅ سيناريو 4: الإلغاء والإرسال مجدداً
```
1. المستخدم يبدأ إرسال ملف
2. يضغط زر الإلغاء
3. النظام يلغي الحماية فوراً
4. يمكن بدء إرسال جديد مباشرة
```

## 📝 ملاحظات تقنية

### Thread Safety
- الحماية تعمل في main thread
- استخدام `QMetaObject.invokeMethod` للتحديثات الآمنة
- لا توجد race conditions

### Memory Management
- تنظيف الملفات المؤقتة تلقائياً
- إلغاء الحماية في جميع الحالات
- لا توجد تسريبات للذاكرة

### Error Handling
- معالجة جميع حالات الفشل
- إلغاء الحماية عند الأخطاء
- رسائل خطأ واضحة
